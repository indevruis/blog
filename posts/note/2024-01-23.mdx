2024년에는 어떤 것을 하면 좋을까 고민하던 때에 블로그를 직접 만들어서 글을 써보는 게 어떨까? 하는 생각이 들었습니다.

2023년 FEConf에 참여했을 때 생각보다 많은 분이 직접 블로그를 만들어 포스트를 작성하고 계시더라고요. ~~(솔직히 말하면 '간지' 나서 도전하는 겁니다.)~~

아직 말하는 감자인 저로서는 빨리 만들지는 못해도 꾸준히 만드는 것을 목표로 블로그를 만들기 시작했습니다.(1일 1커밋이 목표였습니다(੭\*ˊᵕˋ)੭)

---

## 블로그 제작 전

### 1. 기술 스택 선정하기

저는 먼저 기술 스택을 선정했습니다.

아직 UI 만드는 것만 잘하지만.. Next.js와 TypeScript를 사용해 보고 싶었기에 Next.js(App Router), TypeScript, tailwindCSS로 블로그를 만들기로 하였습니다.

&nbsp;

### 2. UI 디자인 하기

디자인은 너무나도 어려운 세계이기 때문에 아래의 예쁜 블로그(+ velog)를 참고하였습니다. 🙇🏻‍♀️

(디자인은 간단한 기능을 먼저 만들어 보고 바꾸자고 생각했기 때문에 차차 수정할 예정입니다.)

- [https://bepyan.me](https://bepyan.me, 'bepyan blog link')
- [https://miryang.dev/blog](https://miryang.dev/blog, 'miryang blog link')
- [https://velog.io](https://velog.io/, 'velog')

&nbsp;

### 3. 필요한 기능 생각하기

간단한 블로그 만들기밖에 해보지 못해서 필요한 기능이 어떤 것이 있을지 고민해 보았습니다.

(사실 처음에 당연히 게시글을 작성하려면 디비를 써야겠구나 했던.. 감자였습니다.)

페이지

- 홈페이지

- 나를 소개하는 페이지

기능

- TOC(Table of Contents)
- Darkmode
- ...

&nbsp;

### 4. 배포 방법

Github Actions도 생각해 보았지만 Next.js를 사용하고 있으니 [vercel](https://vercel.com/new/indevruis-projects, 'vercel page')로 배포하기로 했습니다.

> Vercel의 프런트엔드 클라우드는 개발자에게 더욱 빠르고 개인화된 웹을 구축할 수 있는 프레임워크, 워크플로 및 인프라를 제공합니다.
